Video 2 phút: dùng MXD Importer để tải ảnh + JSON lên repo

Nếu bạn thích xem làm thật trước khi đọc chữ: xem video này. Trong video, Mai minh họa trực tiếp cách dùng tool MXD Importer v4.4.0 PRO để convert ảnh, sinh JSON và upload lên repo.

Link trực tiếp trên YouTube: https://youtu.be/ueYM83ty4AY

1. Ảnh loại nào thì nằm ở thư mục nào?

Để tool không lộn xộn, MXD quy ước rõ:

  • Ảnh sản phẩm (thẻ sản phẩm, g.html, store…)
    → /assets/img/products/
    → tên file = SKU + .webp
    Ví dụ: /assets/img/products/may-khoan-bosch-gsb13.webp
  • Ảnh đại diện bài blog / AFF Academy
    → /assets/img/tin-tuc/
    Ví dụ: /assets/img/tin-tuc/lo-trinh-webshop-affiliate.webp
  • Ảnh OG khi share Facebook/Zalo
    → /assets/img/og/
    Ví dụ: /assets/img/og/dang-ky-mxd.webp
  • Ảnh icon danh mục (hình tròn trong store, quick-nav)
    → /assets/img/categories/
    Ví dụ: /assets/img/categories/do-gia-dung.webp

Quy tắc đơn giản: sản phẩm → /products/, bài/landing → /tin-tuc/ hoặc /og/, danh mục → /categories/.

2. Điền tên sản phẩm & tạo SKU tự động

Giả sử bạn đã có ảnh gốc của 1 sản phẩm muốn đưa lên Webshop.

2.1. Điền tên sản phẩm

  1. Mở ảnh sản phẩm trên máy để nhớ tên / loại sản phẩm.
  2. Trong block 3) Thông tin sản phẩm, ô Tên sản phẩm: nhập tên rút gọn, ví dụ:
    Máy khoan Bosch GSB13 RE 650W – Hàng chính hãng

2.2. Bấm “Điền auto từ tên” để tạo SKU

  1. Nhìn ngay dưới ô Tên sản phẩm, bấm nút “Điền auto từ tên”.
  2. Tool sẽ tự điền vào ô SKU (bắt buộc cho publish; có thể trống khi chỉ upload ảnh) một chuỗi không dấu, nối bằng gạch ngang, ví dụ:
    may-khoan-bosch-gsb13-re-650w
  3. Từ đây, SKU chính là “mã chuẩn” để:
    • Đặt tên file ảnh: SKU.webp.
    • Điền vào trường sku trong JSON.

3. Dùng MXD Importer để tạo ảnh WebP đúng chuẩn

3.1. Cấu hình block “2) Ảnh sản phẩm”

  1. Trong tool, tới phần 2) Ảnh sản phẩm (đơn lẻ).
  2. Định dạng xuất: chọn webp.
  3. Preset kích thước: chọn Vuông (ví dụ 1200×1200). Nếu không có preset, đặt:
    • Max side / Width: 1200
    • Height: 1200
  4. Chất lượng: kéo khoảng 0.9–0.95 cho nét.
  5. Thư mục mặc định:
    • Nếu là ảnh sản phẩm: chọn /assets/img/products/.
    • Nếu là ảnh bài blog: /assets/img/tin-tuc/.
    • Nếu là ảnh OG khi share: /assets/img/og/.
  6. Đảm bảo ô Đường dẫn tuỳ chỉnh kết thúc bằng dấu /, ví dụ: /assets/img/products/.
  7. Tích chọn “Dùng tên file SKU trống” → để nếu ô “Ảnh (đường dẫn)” đang trống, tool sẽ đặt tên file ảnh theo SKU.

3.2. Kéo ảnh vào tool & convert

  1. Ở khung lớn trong block 2), kéo thả ảnh gốc từ máy vào (hoặc bấm Chọn ảnh rồi chọn file).
  2. Kiểm tra lại: định dạng = webp, kích thước vuông, thư mục đúng.
  3. Bấm nút Convert / Nén:
    • Tool sẽ xử lý ảnh, cắt/cân về đúng kích thước & nén.
  4. Nếu muốn lưu ảnh về máy: bấm nút Tải ảnh (browser sẽ tải file về).
  5. Nếu muốn upload thẳng lên repo (khi tool đã kết nối Worker): bấm nút Upload ảnh → /assets/img/….
  6. Có nhiều ảnh muốn xử lý một lượt: sau khi convert xong mỗi ảnh, bấm Thêm ảnh hiện tại vào hàng đợi, cuối cùng mới upload cả hàng đợi.

4. Điền đúng đường dẫn ảnh & sinh JSON sản phẩm

4.1. Điền đường dẫn ảnh

  1. Quay lại block 3) Thông tin sản phẩm.
  2. Ô Ảnh (đường dẫn):
    • Nếu đã bật “Dùng tên file SKU trống”, tool thường tự điền: /assets/img/products/<SKU>.webp.
    • Nếu chưa tự điền, bạn gõ tay: /assets/img/products/may-khoan-bosch-gsb13-re-650w.webp.
  3. Kiểm tra:
    • Tên file = đúng SKU + .webp.
    • Đường dẫn đúng thư mục (products / tin-tuc / og / categories).

4.2. Sinh JSON sản phẩm

  1. Sau khi đã điền xong:
    • Tên sản phẩm
    • SKU
    • Giá
    • Merchant (shopee, lazada, tiki, tiktok...)
    • Link gốc (origin_url)
    • Ảnh (đường dẫn)
    • Danh mục, brand, featured, status…
  2. Nhấn nút Thêm sản phẩm / + Thêm để đưa vào danh sách.
  3. Sau khi đã thêm đủ 10–20 sản phẩm, nhấn nút Xuất JSON / Generate affiliates.json.
  4. Tool sẽ:
    • Hiển thị khối JSON trên màn hình hoặc
    • Tải file affiliates.json về máy.
  5. Nếu chỉ hiển thị trên màn hình: bấm Copy hoặc bôi đen > Copy.

5. Upload ảnh + JSON lên repo & những lỗi thường gặp

5.1. Upload ảnh lên GitHub

  1. Mở repo Webshop trên GitHub.
  2. Vào thư mục ảnh đúng với loại ảnh:
    • Sản phẩm: /assets/img/products/
    • Bài blog: /assets/img/tin-tuc/
    • OG: /assets/img/og/
    • Danh mục: /assets/img/categories/
  3. Nhấn Add file → Upload files.
  4. Kéo các file WebP đã convert từ máy vào > kéo xuống dưới, điền ghi chú commit ngắn (ví dụ: add product images) > bấm Commit changes.

5.2. Dán JSON vào affiliates.json

  1. Trong repo, vào /assets/data/affiliates.json.
  2. Nhấn cây bút Edit this file.
  3. Chọn Select all > xóa nội dung cũ (hoặc dán thêm vào đúng vị trí nếu bạn đã biết cách ghép).
  4. Dán khối JSON mới từ MXD Importer vào.
  5. Cuộn xuống cuối > ghi chú commit (ví dụ: add 20 new products) > bấm Commit changes.
  6. Đợi vài phút cho GitHub build lại, sau đó:
    • Mở /store.html và trang danh mục liên quan.
    • Kiểm tra: sản phẩm hiện đúng, ảnh đúng, nút mua chạy sang sàn.

5.3. Những lỗi thường gặp & cách gỡ

  • Ảnh 404:
    • Quên upload ảnh lên đúng thư mục.
    • Tên file ảnh khác SKU (ví dụ SKU khác 1–2 ký tự).
  • Trang trắng / không hiện sản phẩm:
    • JSON sai dấu phẩy, dấu ngoặc (thiếu , hoặc }).
    • Dán thêm sản phẩm vào nhưng quên thêm dấu phẩy giữa các item.
  • Nút mua không chạy isclix:
    • Merchant trong JSON không đúng list (ví dụ shope thay vì shopee).
    • Link origin không phải link sản phẩm thật (copy nhầm).
